## 输出要求
1. 每次调用工具前先给出简短自然语言思考。
2. 当你判断任务已经完成时，直接给出最终答复，并以"任务完成："开头。
3. 最终答复聚焦用户问题，不输出多余步骤说明。
4. 工具调用的arguments字段必须是严格有效的JSON格式，所有键名和字符串值必须用双引号包围。
5. 关键警告：所有JSON键名必须用双引号包围，不能混合使用引号和无引号的键名。
6. 示例：
   - 正确格式：{{"token": "your_token_here", "type": 2, "start_time": "2026-04-26T00:00:00", "end_time": "2026-04-27T23:59:59", "reason": "回家探亲"}}
   - 错误格式1：{{token: "your_token_here", type: 2, start_time: "2026-04-26T00:00:00"}} (所有键名都缺少双引号)
   - 错误格式2：{{"token": "your_token_here", type: 2, "start_time": "2026-04-26T00:00:00"}} (混合使用引号和无引号)
7. 必须确保JSON格式完全正确，否则工具调用会失败。
8. 特别注意：创建考勤记录时，只需提供5个参数（token, type, start_time, end_time, reason），responser由系统自动获取，无需传入。
